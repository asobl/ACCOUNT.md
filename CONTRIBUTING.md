# Contributing to ACCOUNT.md

Thank you for your interest in improving the ACCOUNT.md specification.

## How to Contribute

1. **Open an issue first.** Before submitting a pull request, open an issue describing what you want to change and why. This prevents duplicate work and lets the community weigh in on the direction.

2. **Fork and branch.** Fork the repository, create a branch from `main`, and make your changes there.

3. **Submit a pull request.** Reference the issue number in your PR description. Keep changes focused on one thing at a time.

## What We're Looking For

- **Spec improvements:** Clearer language, better examples, edge cases we missed.
- **New examples:** ACCOUNT.md files for different industries or team sizes. Examples should use fictional companies with realistic data.
- **Tooling:** Scripts or CLI tools that work with the format (linting, health reports, cross-account queries).
- **Bug reports:** If something in the spec is ambiguous, contradictory, or broken in practice, tell us.

## Guidelines

- Keep examples vendor-neutral. No real company names or data.
- Follow the existing spec structure when proposing changes.
- One PR per change. Don't bundle unrelated improvements.

## Code of Conduct

Be respectful. Give constructive feedback. Assume good intent.
